## What this is

Degen Dogs Mission 3 Analytics is a cached static dashboard for the Mission 3 auction
era on Base. It reads public onchain auction data, runs the approved local SQL analytics
layer, exports CSV/JSON datasets, and publishes a GitHub Pages site that community
members can inspect without running a database.

The project is independent community infrastructure. It is built around Degen Dogs,
which was created by Mark Carey / dogmaster, but it should not be described as an
official app unless the Degen Dogs project explicitly approves that wording.

## Why it exists

Degen Dogs needed a project-specific dashboard and archive layer that is easy to
inspect, fork, rebuild, and share. This repo keeps the data pipeline, SQL query layer,
generated exports, dashboard shell, and historical archive work together so the
community can audit how the public snapshot is produced.

- downloadable CSV/JSON exports for published tables.

## Architecture

The dashboard is static, but the data is generated by a local runner before publishing.

```text
Base RPC + contract calls
  -> local runner / Mac mini
  -> Python decoder + in-memory SQLite
  -> sql/mission3_dashboard.sql
  -> generated CSV/JSON + index.html
  -> GitHub Pages dashboard
```

Visitors do not run SQL in the browser. The local pipeline fetches public/onchain data,
executes approved SQL, writes static artifacts, and GitHub Pages serves the built site.

See [`docs/architecture.md`](docs/architecture.md) and
[`docs/data-pipeline.md`](docs/data-pipeline.md) for details.

## Quick start

```bash
npm ci
npm run data
npm run dev
npm run build
```

- `npm run data` refreshes public/onchain data and regenerates `generated/`,
  `public/generated/`, `index.html`, and `README.md`.
- `npm run dev` previews the dashboard locally.
- `npm run build` creates the production build in `dist/`.

## Trust and caveats

- This is an independent community-built analytics/archive contribution.
- The public dashboard is a cached snapshot, not a real-time database.
- Current auction state comes from onchain contract calls.
- Historical rows are reconstructed from event logs and checked-in archive outputs.
- Farcaster profile resolution can be incomplete.
- Reward estimates are dashboard context, not official accounting.
- Mission 1 and Mission 2 archive coverage is documented honestly in `archive/` and may
  differ by era.

Read [`docs/trust-and-caveats.md`](docs/trust-and-caveats.md) before using the data for
decisions.
