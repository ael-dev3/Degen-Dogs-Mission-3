# Degen Dogs Mission 3 Archive

Mission 3 is the Base era of Degen Dogs auctions. This folder is the active rolling
archive for Mission 3 onchain auction data.

The archive is independent and community-built. It is designed to support the existing
Mission 3 dashboard, future Dog search/archive features, and long-term reproducibility.

## What is archived

- Raw auction-house logs from Base.
- Decoded `AuctionCreated`, `AuctionBid`, `AuctionExtended`, and `AuctionSettled` event
  rows.
- Current auction snapshots from the auction-house contract.
- SQLite database tables for raw/decoded data.
- Generated CSV/JSON outputs and manifest files.
- Public-ready JSON for future dashboard search/archive features.

## How to rebuild

```bash
npm ci
npm run archive:mission3:verify
npm run archive:mission3:full
npm run archive:mission3:index
```

The full run rebuilds `archive/mission3/data/mission3_archive.sqlite` from the verified
Mission 3 start block. The incremental run starts from the latest indexed block stored
in `mission3_index_state`.

## How incremental indexing works

1. Read verified config from `archive/mission3/config/`.
2. Open or create `archive/mission3/data/mission3_archive.sqlite`.
3. Determine the next block:
   - `latest_indexed_block + 1` for incremental runs.
   - verified Mission 3 `from_block` for full refreshes.
4. Fetch Base logs in bounded chunks.
5. Store raw logs idempotently by `(chain_id, transaction_hash, log_index)`.
6. Decode event-specific rows idempotently.
7. Refresh derived marts and generated CSV/JSON.
8. Optionally copy small public files to `public/generated/mission3/`.

## Relationship to the live dashboard

The current dashboard still displays Mission 3 only. It is generated by
`scripts/build_dashboard.py` and `sql/mission3_dashboard.sql`.

The archive indexer is separate so archive work does not break the existing static
dashboard. A future dashboard pass can safely read from the archive-generated search and
metrics JSON.

## Reward estimate provenance

The public dashboard may show per-Dog WOOF/SUP reward estimate tiles. The repo documents
that these estimates use a 141-Dog reward-basis snapshot and intentionally exclude WOOF
Vault Bonus. Treat reward values as estimates unless confirmed against official reward
logic.
