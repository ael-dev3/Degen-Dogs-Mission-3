# Reconstruction / Backup Runbook

This folder explains how to rebuild the Degen Dogs Mission 3 dashboard if the current
local runner goes offline, disappears, breaks, or cannot push updates.

The dashboard is static, but the data is generated by a local runner. A new maintainer
can reconstruct the pipeline by forking the repo, configuring a Base RPC, running the
data script, validating outputs, committing generated files, and enabling GitHub Pages.

## Most important recovery fact

GitHub Pages builds the static site, but it does not fetch fresh chain data. If a
maintainer wants fresh data in a fork, they must run `npm run data` locally, validate
the results, commit the regenerated files, and then push.
